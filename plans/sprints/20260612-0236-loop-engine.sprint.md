## PRD

### Problem

- loop-in-hook 的 file-backed 脊柱与 edit/done 层确定性裁决已被 NLAH 论文(arXiv 2603.25723)定量背书(file-backed state +1.6/+5.5,acceptance-tightening +4.8),但三个结构性缺口未被覆盖:
  1. **反模式**:~600 行 TS prompt-intent 分类器用代码推断自然语言语义,与论文验证的分工(NL 表达 harness 逻辑、in-loop LLM 解释、代码做确定性操作)相反;lessons.md 已记录三轮误报修补,维护无界。
  2. **无委派面**:`tasks/contracts/*.contract.md` 已具备 agent-call κ 的 80%(allowed_paths/verification/exit criteria),缺 budget/permission scope/roles,且只用于约束人类会话,从不作为任务包派给 child agent(论文 Full IHR 下 ~90% 工作在 delegated children)。
  3. **无 heartbeat**:Addy Osmani Loop Engineering 五组件中唯一完全缺失项;发现/分诊全部依赖人开口。
- 本 sprint 按 research report 的 **Staged clean path** 执行:证据先行,每步可回退,cutover 被两道 gate 锁住。

### Users

- 本仓库维护者(self-host dogfooding):日常通过 Claude Code / Codex 在该仓库执行 plan→contract→verify 流程的人。
- 下游 generated repos 的维护者:hook/contract 模板变更最终经 `assets/` 镜像下发(本 sprint 内只动 self-host 面,generated 面同步在 cutover 任务内显式声明)。

### Success Criteria

- Routing A/B eval(TS verdict vs 状态快照+NL 决策表自路由)在 Claude+Codex 各完成一轮非 dry-run,合规率/误报/token 增量数据落盘——**无论结果 go 还是 no-go,sprint 都算产出了决定性证据**。
- go 路径:shadow 期 divergence 数据达标后,TS 语义分类删除,`prompt-guard.sh` 从 1,076 行降至 ~300 行以内,hook-runtime phase-probe 计时不高于基线。
- no-go 路径:分类器收缩为显式确定性触发集合,决策表仍迁入 NL 契约文本作为 advisory,sprint 在 Execution Log 记录改道理由。
- contract κ 字段(budget/permission_scope/roles)向后兼容落地,且至少 1 个真实 contract 通过 `contract-run` 由 worker/verifier child 对完成(verifier rubric == contract exit criteria)。
- heartbeat v0 连续 3 次定时运行产出 triage inbox 条目;两周后采纳率复盘写入 Execution Log。

### Acceptance Scenarios

- 维护者新增一条路由规则:只编辑 NL 决策表文本,不改 TS、不发版,下一个会话即生效(cutover 后)。
- 维护者对一个 Approved plan 运行 `contract-run`:worker child 在 contract worktree 内完成实现,verifier child 依 exit criteria 写出 review 文件,done gate 接受该 review,全程 parent 只 narrate。
- 维护者早晨打开 triage inbox,看到一条由定时任务发现(而非人触发)的待处理项(drift request / 工作流检查失败 / sprint next 建议)。
- 任一 gate 失败时:回退动作只需移除注入/还原模板,TS 路径在 shadow 期内始终保持权威,无需 revert 产品行为。

### Non-goals

- 不做 multi-candidate search、不给 verifier 发明独立于 contract exit criteria 的 rubric、不默认重型编排(NLAH 负面结果:verifier −8.4、multi-candidate dominated、Full IHR 13× tokens)。
- 不自动化 plan 人工批准与外部 acceptance(supervision 边界,"stay the engineer")。
- 不动 file-backed 脊柱(plans/contracts/checks/handoff)与 edit 层确定性裁决。
- 不在本 sprint 内重写 generated-repo 升级/迁移工具链;cutover 任务只负责把变更同步进 `assets/` 镜像并标注 self-host/generated 影响面。

## Architecture Notes

### Capabilities Touched

- `runtime-harness/hook-adapters`(`.ai/hooks/`,prompt-guard、stop-orchestrator、observer)
- `src/cli/hook/`(prompt-intents.ts、prompt-guard-decision.ts、route registry)
- 契约面:contract 模板、`scripts/plan-to-todo.sh`、`scripts/verify-contract.sh`、`scripts/contract-worktree.sh`
- 评测面:`bun run benchmark:skills`(既有 `route-workflow-check` eval 基建)
- 文档面:workflow contract / `docs/reference-configs/`(NL 决策表落点)、README hook 流程图(cutover 时)

### Dependency Order

- **Track A(分类器替换主线,串行)**:01 → 02(**gate G1:go/no-go**)→ 03(shadow)→ 07(**gate G2:divergence 达标后 cutover**)
- **Track B(委派,串行,独立于 A)**:04 → 05
- **Track C(heartbeat,独立)**:06
- **Stretch**:08(hook diet,依赖 07 完成后的拓扑稳定)
- G1 no-go 时:03/07 替换为"分类器收缩为显式触发"改道任务,Track B/C 不受影响。

### Risks

- **NL 自路由合规率不足(G1 失败)** → 改道:保留确定性显式触发,决策表仅作 advisory;已在 backlog 设计中预留。
- **快照+NL 表 token 增量失控** → 快照降频/瘦身;eval(02)显式度量 token 增量,超出 advisory 现状一个量级即判 no-go。
- **contract-run 子代理 review 质量劣于单会话基线** → 05 试点对照记录;劣化则回退为"同会话角色分离"弱形态,07 不受影响。
- **heartbeat 无人采纳** → 两周复盘为 falsifier;无采纳即撤 cron,06 标记为 reverted 而非强行保留。
- **shadow 期双轨成本**(TS verdict + 快照同时运行)→ 时间盒:shadow 期以 2 周或 ≥100 条真实 prompt 为上限,到期必须裁决。
